安装Gulp

Gulp.js 是一个自动化构建工具,开发者可以使用它在项目开发过程中自动执行常见任务。

先要全局安装gulp：

然后还需要在项目中单独安装一次

使用

1.新建一个gulpfile.js文件，输入内容;

2.运行gulp任务;
切换到存放gulpfile.js文件的目录；
命令行中执行gulp命令，gulp后面可以加上要执行的任务名，例如gulp task1;
如果没有指定任务名，则会执行任务名为default的默认任务；

API

gulp.task(name[, deps], fn)

定义任务

参数name 为任务名；
deps 是当前定义的任务需要依赖的其他任务，为一个数组。当前定义的任务会在所有依赖的任务执行完毕后才开始执行。如果没有依赖，则可省略这个参数；
fn为任务函数，可选参数，我们把任务要执行的代码都写在里面。
如，定义一个简单的任务

gulp.src(globs[, options])

选择文件

globs参数是文件匹配模式(类似正则表达式)，用来匹配文件路径(包括文件名)，也可以直接指定某个具体的文件路径，当有多种匹配模式时可以使用数组；options是可选参数；

gulp.dest(path)

输出文件，传入参数是输出文件的路径。

gulp.dest(path)生成的文件路径是传入的path参数后面再加上gulp.src()中有通配符开始出现的那部分路径。例如：

gulp.pipe()

管道，可以理解为将操作加入执行队列。

gulp.watch()

监视文件的变化

第一种使用方式

gulp.watch(glob[, opts], tasks);
参数tasks 为文件变化后要执行的任务，为一个数组；

第二种使用方式

gulp.watch(glob[, opts, cb]);
cb参数为一个函数。每当监视的文件发生变化时，就会调用这个函数，并且会给它传入一个对象，该对象包含了文件变化的一些信息，type属性为变化的类型，可以是added，changed，deleted；path属性为发生变化的文件的路径。

插件使用

使用gulp压缩JS

目的：找到js/目录下的所有js文件，压缩它们，将压缩后的文件存放在dist/js/目录下。并且当js文件修改后，能检测代码修改自动执行任务；

安装gulp-uglify模块：npm install gulp-uglify；在你的文件夹中会新增一个node_modules文件夹，这里面存放着npm安装的模块。
在gulpfile.js中编写代码

在命令行中输入gulp script执行任务；
执行任务后，我们可以在目录dist/js/a.js找到压缩后的js文件；
js/a.js一旦有修改 就必须重新在命令行输入gulp script，这很麻烦。所以要检测代码修改自动执行任务；可以使用 gulp.watch()来实现；

在命令行输入gulp auto；会自动监听js/*.js文件的修改后压缩js。修改js/a.js中的代码并保存，命令行将会出现提示，表示检测到文件修改并压缩文件。
使用gulp.watch后命令行会进入“运行”状态，此时不可以在命令行进行其他操作。可通过Ctrl + C停止 gulp。
